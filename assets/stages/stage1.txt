; Stage 1 map — see docs/stage-authoring.md.
; Grid: 1 char = 32 px. Columns = scroll distance, rows = lanes (11 rows for
; the 352px play area, top to bottom). A glyph triggers when its column
; reaches the right screen edge (~0.8s of scroll per column at 40 px/s).
; Blocks: `@<px>` header, then exactly 11 grid rows. Blocks are contiguous;
; the boss lock fires when the map ends (@7200).
; Lines starting with `;` are comments (`#` is the land glyph, not a comment).
;
; LEGEND
;   air (spawn events):                     surface (world-anchored):
;     d  lone Skimmer Drone (sine)            C  Casemate        m  Mine
;     w  line of 3 drones                     T  Tracking Turret
;     W  V of 5 drones (leader anchored)      R  Relay Node
;     i  Interceptor (straight, fires once)   P  Mobile Platform
;     G  Gunship (3-way spread)
;     b  Skimmer Drone flanker: spawns off the LEFT edge flying right,
;        the mirror of every other air glyph - occasional, not a new
;        pattern to sweat over every beat.
;   land installations (Stage 2+; the glyph's cell compiles as terrain
;   + hardpoint pad + spawn, so the pad always has something on it):
;     B  Mortar Battery   K  Drone Bunker
;   terrain:
;     #  land footprint (blocks torpedoes; contiguous runs merge)
;     H  land hardpoint (terrain cell with a fortified mounting pad).
;        Author one ONLY under an installation that actually mounts
;        there: an empty "could hold a turret" pad is visual noise, so
;        Stage 1 has none (B/K glyphs make bare H mostly unnecessary).
